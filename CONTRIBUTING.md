# Contributing to AEON Universal Agents

Thank you for helping improve AEON Universal Agents.

This project is intentionally simple: it is a universal instruction standard for coding agents. Contributions should make agents more objective, safer, clearer, and more useful across many codebases.

---

## Contribution principles

Good contributions should:

- improve objectivity
- reduce hallucination risk
- reduce unnecessary code
- improve repository pre-flight behavior
- improve documentation discipline
- improve testing and verification discipline
- improve frontend quality guidance
- improve clean architecture and MVC responsibility separation
- remain stack-neutral and community-ready

Avoid contributions that:

- make the file specific to one company or product
- force one framework, programming language, or runtime
- add hype language
- make the file unnecessarily long
- encourage agents to claim unverified facts
- encourage agents to rewrite working systems without need

---

## How to contribute

1. Fork the repository.
2. Create a branch.
3. Make a focused change.
4. Keep wording direct and practical.
5. Open a pull request explaining:
   - what changed
   - why it improves agent behavior
   - whether it affects universality

---

## Style guide

Use:

```txt
short sentences
clear rules
practical checklists
objective wording
agent-actionable instructions
```

Avoid:

```txt
marketing language
long theory sections
project-specific assumptions
framework bias
company-specific rules
```

---

## Review checklist

Before submitting, check:

```txt
Does this help most developers?
Is it stack-neutral?
Does it reduce hallucination risk?
Does it avoid unnecessary length?
Does it preserve the core command?
Is it easy for an agent to follow?
```
